# 연수 참석 등록 시스템

부산남부교육지원청 대강당에서 열리는 직원 연수의 출석을 **로그인 없이**, 그러나 **부정 참석은 차단하면서**, **인사 명단과 정확히 매칭**되도록 등록하기 위한 모바일 웹 시스템입니다.

종이 명단 서명 방식의 세 가지 고질적인 문제 — (1) 대신 등록(부정 출석), (2) 이름 오기재·동명이인 혼선, (3) 종료 후 수기 입력 부담 — 을 한 번에 해결하는 것을 목표로 합니다.

---

## 핵심 아이디어

| 문제 | 해결 방식 |
| --- | --- |
| 대강당에 오지 않고 부정 등록 | **현장 PIN(기본 5분 자동 갱신)** 입력 + PIN 일련번호·발급/만료 시각 기록. 만료 PIN은 EXPIRED로 거절. |
| 이름 오기재·동명이인 혼선 | 자유 입력 제거. 인사 명단을 **사전 적재** 후 "부서 → 이름" 2단계 선택. 동명이인은 직급과 사번 끝 4자리를 함께 표시. |
| 종이 명단 후처리 | 실시간 등록률·미등록자 모니터링 + **CSV/Google Sheets** 즉시 산출. |
| 별도 앱 설치 | 단일 모바일 웹페이지(QR 안내). 별도 서버 없이 **Google Sheets**를 백엔드로 사용 가능. |

---

## 부정 참석 방지 — 4중 방어선

1. **현장 PIN(시간제한)** — 관리 단말이 4자리 PIN을 표시하고 1·3·5·10분 중 선택한 주기로 자동 갱신. 외부에 공유해도 곧 무효가 됩니다.
2. **PIN 발급 단말 단일화** — 관리 페이지는 별도 비밀번호로 보호하며, 담당자 단말 1대에서만 운영합니다.
3. **1인 1회 등록(DUP 차단)** — 동일 사번의 재등록은 거부.
4. **감사 로그** — 등록 시각, PIN 일련번호, 단말 핑거프린트를 함께 기록해 사후 감사가 가능합니다.

이 시스템은 단일 수단으로 100% 차단을 보장하지 않습니다. **짧은 갱신 주기 + 단일 발급 단말 + 감사 로그**로 부정의 비용을 충분히 높여 억제하는 모델입니다. 추가 강화가 필요하면 GPS 보조, 내부 Wi-Fi SSID 검증, 시작·종료 이중 PIN 옵션을 결합할 수 있습니다.

## 화면 구성

### 참석자 등록 화면 (모바일)
1. 부서 선택 (드롭다운, 검색 가능)
2. 본인 이름 선택 (해당 부서 명단, 동명이인은 직급·사번 후미 표시)
3. 4자리 현장 PIN 입력
4. "등록" 클릭 → 등록 완료 화면(이름·시각·PIN 일련번호)

### 연수담당자 화면
- 현재 PIN 카드 (대형 표시, 남은 시간 게이지, 갱신 주기 설정)
- 등록 통계 (전체/등록/등록률)
- 실시간 등록자 목록
- 미등록자 검색 (호명 확인용)
- CSV 내보내기 / 미등록자 명단 CSV / 데이터 초기화

## 데이터 구조 (Google Sheets)

### 시트 ① `명단` (사전 적재)
| emp_id | dept | name | rank | active |
| --- | --- | --- | --- | --- |
| 20240137 | 행정지원과 | 홍길동 | 주무관 | TRUE |

### 시트 ② `출석`
| 컬럼 | 예시 | 설명 |
| --- | --- | --- |
| session_id | 2026-05-12-PM | 연수 차시 식별자 |
| emp_id | 20240137 | 명단 시트의 사번 참조 |
| name / dept / rank | 홍길동 / 행정지원과 / 주무관 | 명단에서 자동 채움 |
| registered_at | 2026-05-12 10:14:23 | 등록 시각 |
| pin_serial | P-007 | 어느 PIN으로 등록되었는지 |
| client_fp | a8c3... | 단말 핑거프린트(감사용) |
| status | OK / DUP / EXPIRED | 처리 결과 |

### 시트 ③ `PIN 로그`
PIN 일련번호, 값, 발급/만료 시각, 발급 단말. 감사 종료 후 마스킹 권장.

## 빠른 실행 (데모)

별도 빌드/서버 없이 단일 HTML로 즉시 동작합니다.

```bash
# 저장소 클론 후
open "연수참석등록_데모.html"     # macOS
start "" "연수참석등록_데모.html" # Windows
```

또는 파일을 더블클릭하면 기본 브라우저에서 열립니다.

**시연 절차**
1. "연수담당자 화면" 탭 → 잠금 비밀번호 `1234` 입력 → 현재 PIN 확인.
2. 다른 창(또는 다른 기기에서 같은 파일)에서 "참석자 등록 화면" 진입.
3. 부서 → 이름 → 현재 PIN 입력 → 등록.
4. 담당자 화면에서 등록률과 최근 등록자 목록이 즉시 갱신됩니다.
5. **부정 차단 검증**: 담당자 화면에서 "PIN 즉시 갱신"을 누른 직후, 이전 PIN으로 등록을 시도하면 `EXPIRED` 로 거절됩니다.
6. **중복 차단 검증**: 같은 사람으로 두 번 등록을 시도하면 `DUP` 로 거절됩니다.

데이터는 브라우저 LocalStorage에 저장되며, "데이터 초기화"로 사전 명단을 제외한 모든 등록·PIN 이력을 삭제할 수 있습니다.

## 운영 환경 연동 (Google Apps Script)

데모의 LocalStorage 저장 부분만 `fetch(POST)` 호출로 바꾸면 Google Sheets 백엔드 시스템이 됩니다.

```javascript
// 데모 코드의 save(LS_REG, log) 자리에 다음을 호출
await fetch(GAS_URL, {
  method: 'POST',
  body: JSON.stringify(entry),
});
```

Google Apps Script `doPost` 예시:

```javascript
function doPost(e) {
  const data = JSON.parse(e.postData.contents);
  const sh = SpreadsheetApp.openById(SHEET_ID).getSheetByName('출석');
  sh.appendRow([
    data.session_id, data.emp_id, data.name, data.dept, data.rank,
    new Date(data.registered_at), data.pin_serial, data.client_fp, data.status,
  ]);
  return ContentService.createTextOutput(JSON.stringify({ ok: true }))
    .setMimeType(ContentService.MimeType.JSON);
}
```

PIN 검증을 서버 측에서 하려면 `현재 PIN`도 시트(또는 Script Properties)에 두고 `doPost`에서 일치/만료를 확인하도록 확장합니다.

## 운영 절차 체크리스트

**연수 전 (D-1 ~ 시작 30분 전)**
- 인사 명단 시트 최신화 (인사이동 반영)
- 해당 차시 `session_id` 발급, 연수명·일시 기재
- 대강당 입구·스크린에 등록 QR 게시 위치 점검
- 관리 페이지 명단 동기화 + 테스트 등록 1건 후 삭제

**연수 중**
- 관리 페이지 PIN 카드만 스크린에 미러링
- 등록률 80% 도달까지 추가 안내 멘트
- 종료 직전 미등록자 호명

**연수 후**
- CSV 내보내기 → 행정 시스템(나이스/연수원)에 업로드
- 이상 등록(DUP/EXPIRED)은 사유와 함께 별지 보관
- PIN 로그는 1개월 후 마스킹/삭제

## 한계와 권고

- **PIN 즉시 유출 시나리오**는 본 시스템만으로 100% 차단되지 않습니다. 짧은 갱신 주기 + 단말 핑거프린트 + 감사 로그로 *비용을 높이는* 모델입니다. 더 강한 차단이 필요하면 시작·종료 이중 PIN, 내부 Wi-Fi SSID 제한을 결합하세요.
- **실내 GPS**는 정확도 한계로 단독 차단 수단이 될 수 없습니다. 보조 신호로만 사용합니다.
- **Google Sheets 백엔드**는 100~500명 규모에 충분합니다. 대규모(수백~수천 동시 등록) 행사에서는 Firebase 등 전용 백엔드를 권장합니다.
- **개인정보**는 직무연수 행정 처리 외 목적으로 사용하지 않으며, 차시 종료 후 보존 기간을 1년 이내로 운영합니다.

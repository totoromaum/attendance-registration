# 현장 참석 등록 시스템

강당·회의실·행사장 등 **어느 장소에서든** 쓸 수 있는 현장 참석 확인 시스템입니다. 로그인 없이, 그러나 부정 참석은 억제하면서, 사전 명단과 정확히 매칭되도록 등록하는 모바일 웹입니다.

종이 명단 서명 방식의 세 가지 고질적인 문제를 겨냥합니다.

| 문제 | 해결 방식 |
| --- | --- |
| 현장에 오지 않고 대리 등록 | 현장 화면에만 표시되는 **6자리 인증코드 + QR**(1·3·5·10분 주기 자동 갱신) + 코드 회차 기록 + 기기당 1회 등록 제한 + (선택) **입장·퇴장 이중 확인** + (선택) GPS 반경 확인 |
| 이름 오기재·동명이인 혼선 | 자유 입력 제거. 명단을 사전 적재하고 "부서 → 이름" 2단계 선택. 동명이인은 직급·사번 끝 4자리를 함께 표시 |
| 종료 후 수기 정리 부담 | 실시간 등록률·미등록자 확인, 등록자/미등록자/전체 시도 로그 CSV 즉시 내보내기 |

행사명·장소명·행사장 좌표·허용 반경·명단은 전부 운영자 화면에서 설정합니다. 소스 코드에 특정 기관 정보나 실명 명단이 들어가지 않습니다.

---

## 화면 구성

### 참석자 화면 (모바일)
1. 행사장 스크린의 **QR을 카메라로 스캔** → 등록 페이지가 열리고 인증코드가 자동 입력됨 (직접 입력도 가능)
2. 부서 → 이름 선택 (동명이인은 `이름 · 직급 · 사번 끝 4자리`로 구분)
3. (위치 확인을 켠 경우) 현재 위치 확인
4. "참석 등록" → **출석 확인증**(이름·시각·코드 회차·기기 식별)에 출석 도장이 찍힌 완료 화면
5. (퇴장 확인을 켠 경우) 행사 종료 시 같은 방법으로 **퇴장 확인** → 퇴장 도장이 찍힌 확인증

### 운영자 화면 (PIN 보호)
- **현장 인증코드 보드** — 대형 코드 + 스캔용 QR, 회차 번호(C-001…), 남은 시간 게이지, 갱신 주기 설정, 즉시 재발급. 이 화면만 행사장 스크린에 미러링합니다.
- **실시간 현황** — 대상/입장/퇴장/등록률, 등록자 검색, 미등록자 명단(호명용)
- **예외 기록** — 중복·만료 코드·코드 불일치·위치 이탈·입장기록 없음 등 거절된 시도 전체
- **행사 정보(행사명·장소명·퇴장 확인) / 위치 설정 / 명단 관리 / 내보내기·초기화**

최초 실행 시 운영자 PIN을 직접 설정합니다(소스에 하드코딩된 PIN 없음, 해시로만 저장).

## 부정 참석 방지 설계

1. **시간제한 인증코드 + QR** — 만료되면 자동 재발급. 외부에 유출돼도 곧 무효가 됩니다. 코드 회차가 등록 기록에 남아 어느 코드로 등록했는지 추적됩니다. QR 스캔이 기본 흐름이므로 갱신 주기를 1분까지 줄여도 참석자 부담이 없습니다.
2. **입장·퇴장 이중 확인**(선택) — 운영자가 종료 시점에 "퇴장 확인"으로 전환하면 새 코드가 발급되고, 참석자는 한 번 더 확인해야 합니다. 잠깐 들러 등록만 하고 떠나는 부정 출석의 비용을 크게 높입니다. 퇴장 시각·코드 회차·기기가 함께 기록되며, 입장 기록이 없으면 퇴장 확인이 거절됩니다.
3. **기기당 1회 등록**(기본 켬) — 한 사람이 자기 휴대폰으로 여러 명을 대리 등록하는 경로를 차단합니다. 끄더라도 같은 기기의 다인 등록은 운영자 화면에 경고로 표시됩니다.
4. **1인 1회 등록** — 같은 사번의 재등록은 `중복`으로 거절·기록됩니다.
5. **거절 이력 보존** — 모든 실패 시도(만료/불일치/중복/위치 이탈/입장기록 없음)가 시각·기기와 함께 남아 사후 감사가 가능합니다.
6. **(선택) GPS 반경 확인** — `사용 안 함 / 참고용 / 필수` 3단계. 기본값은 "참고용"입니다.

### 위치(GPS) 확인의 실효성에 대하여

GPS는 강제 수단으로는 약합니다. 실내에서는 오차가 수십 m를 넘어 정당한 참석자가 거절될 수 있고, 브라우저 개발자 도구나 위치 변조 앱으로 좌표를 위조할 수 있으며, 이 데모에서는 검증이 클라이언트에서 일어나므로 우회도 가능합니다. 반면 **기록·증빙 수단으로는 유효**합니다 — 등록 시점의 거리 기록이 남는다는 사실 자체가 부정을 억제하고, 사후 감사 때 이상 등록을 가려내는 근거가 됩니다. 그래서 기본값이 "참고용"이며, "필수"는 야외·소규모 행사에서만 권장합니다. 웹 브라우저에서는 Wi-Fi SSID 확인이나 블루투스 비콘 같은 대안을 쓸 수 없기 때문에, 현장성 검증의 주력은 **짧은 주기의 인증코드(QR)와 입장·퇴장 이중 확인**입니다. 서버 연동 시에는 행사장 Wi-Fi의 공인 IP 확인을 추가할 수 있습니다.

## 빠른 실행 (데모)

별도 빌드/서버 없이 `index.html` 하나로 동작합니다. 파일을 더블클릭해 브라우저에서 여세요.

**시연 절차**
1. "운영자" 탭 → 최초 실행이면 PIN 설정 → 접속
2. "참석 대상 명단" 펼침 → CSV 파일 업로드 또는 "예시 명단 불러오기" → "명단 적용"
3. 인증코드 보드의 6자리 코드 확인
4. "참석 등록" 탭에서 부서 → 이름 → 코드 입력 → 참석 등록
5. 출석 도장이 찍힌 확인증이 표시되고, 운영자 화면 현황이 즉시 갱신됩니다
6. **중복 차단**: 같은 사람으로 재등록하면 거절되고 예외 기록에 남습니다
7. **만료 차단**: "새 코드 즉시 발급" 직후 이전 코드로 시도하면 거절됩니다
8. **기기 차단**: 같은 브라우저에서 다른 사람으로 등록하면 기기 중복으로 거절됩니다
9. **퇴장 확인**: 행사 정보에서 "종료(퇴장) 확인 사용"을 켜고 코드 보드의 "퇴장 확인 시작"을 누르면, 참석자 화면이 퇴장 확인 모드로 바뀝니다. 입장 기록이 있는 사람만 퇴장 확인이 가능합니다
10. **QR 진입**: 호스팅된 주소로 열면 QR 스캔 시 `#c=코드` 해시로 진입해 코드가 자동 입력됩니다 (파일로 직접 열면 QR에 코드 숫자만 담깁니다)

같은 브라우저의 다른 탭·창과는 실시간으로 동기화됩니다(BroadcastChannel).

## 데모의 한계 — 반드시 읽어주세요

이 단일 파일 버전은 **같은 브라우저 안에서 완결되는 데모**입니다. 실제 행사에서 참석자 개인 휴대폰으로 등록을 받으려면 서버 연동이 필요합니다.

- 데이터가 브라우저 LocalStorage에 저장되므로 **다른 기기와 자동 공유되지 않습니다.**
- 코드 검증·중복 판정이 브라우저 안에서 일어나므로, 기술 지식이 있는 사용자는 개발자 도구로 우회할 수 있습니다. 운영자 PIN도 화면 잠금 수준의 보호이지 보안 인증이 아닙니다.
- **부정 방지가 실제로 성립하려면 코드 검증·등록 저장·중복 판정을 서버 측에서 수행해야 합니다.** 아래 연동 가이드를 참고하세요.

## 운영 환경 연동 (Google Apps Script + Sheets)

등록 검증을 서버로 옮기는 최소 구성입니다.

**시트 구성**
- `명단`: `emp_id, dept, name, rank`
- `출석`: `ts, status, emp_id, dept, name, rank, code_serial, distance, device_id`
- Script Properties: 현재 코드(`code`), 회차(`serial`), 만료 시각(`expires_at`)

**Apps Script `doPost` 예시** — 검증을 서버에서 수행:

```javascript
function doPost(e) {
  const req = JSON.parse(e.postData.contents);
  const props = PropertiesService.getScriptProperties();
  const ss = SpreadsheetApp.openById(SHEET_ID);
  const att = ss.getSheetByName('출석');

  let status = 'OK';
  if (Date.now() > Number(props.getProperty('expires_at'))) status = 'EXPIRED';
  else if (req.code !== props.getProperty('code')) status = 'CODE_MISMATCH';
  else {
    const ids = att.getRange('C2:C').getValues().flat();
    if (ids.includes(req.emp_id)) status = 'DUP';
  }

  att.appendRow([new Date(), status, req.emp_id, req.dept, req.name, req.rank,
                 props.getProperty('serial'), req.distance, req.device_id]);
  return ContentService.createTextOutput(JSON.stringify({ status }))
    .setMimeType(ContentService.MimeType.JSON);
}
```

클라이언트에서는 `index.html`의 등록 처리(`btn-submit` 핸들러)에서 로컬 검증 대신 위 엔드포인트로 `fetch(POST)` 하고 응답의 `status`로 완료/거절 화면을 표시하면 됩니다. 코드 발급(운영자 보드)도 같은 방식으로 Script Properties를 갱신하는 엔드포인트를 추가합니다.

Google Sheets 백엔드는 수백 명 규모에 충분합니다. 수천 명 동시 등록 규모라면 Firebase 등 전용 백엔드를 권장합니다.

## 명단 CSV 형식

```
사번,부서,이름,직급
20240101,기획운영과,홍길동,주무관
20240102,기획운영과,김철수,실무관
```

명단을 넣는 방법은 세 가지입니다. 운영자 화면 → "참석 대상 명단"에서:

1. **CSV 파일 업로드** — 파일을 선택하면 바로 적용됩니다. UTF-8과 한글 엑셀 기본 인코딩(EUC-KR/CP949)을 자동 인식합니다.
2. **붙여넣기** — 엑셀에서 범위를 복사해 입력란에 붙여넣어도 됩니다(탭 구분 자동 인식). "명단 적용"을 누르세요.
3. **예시 명단** — 시연용 32명 명단을 불러올 수 있습니다. 부서는 부산남부교육지원청 조직도(유초등교육지원과·중등교육지원과·학생건강지원과·행정과·학교지원과)를 따르고, 이름은 익명(성+○○) 더미 데이터입니다.

- 첫 줄 제목행은 자동 인식되어 건너뜁니다. 직급은 생략 가능합니다.
- 쉼표가 든 값은 따옴표로 감싼 형식(`"기획, 운영과"`)을 지원합니다.
- 같은 사번이 중복되면 첫 항목만 사용합니다.
- 명단은 적용한 기기의 브라우저에만 저장됩니다.

## 운영 절차 체크리스트

**행사 전**
- 운영자 화면에서 행사명·장소명 설정, 최신 명단 CSV 적용
- 위치 확인을 쓸 경우: 행사장에서 "현재 위치를 행사장으로 설정" → 저장
- 입구·스크린에 등록 페이지 QR 게시, 테스트 등록 1건 후 "등록 기록만 삭제"

**행사 중**
- 인증코드 보드만 스크린에 미러링
- 등록률을 보며 추가 안내, 종료 직전 미등록자 명단으로 호명

**행사 후**
- 등록자·미등록자 CSV 내보내기 → 행정 시스템에 업로드
- 예외 기록(전체 시도 로그 CSV)은 사유 확인 후 별도 보관
- 개인정보가 담긴 기록은 보존 기간(권장 1년 이내) 경과 후 삭제

## 기술 스택

- 단일 HTML / Vanilla JS / CSS — 외부 의존성 없음, 내부망·오프라인 동작 (QR 생성기 내장, byte 모드·ECC L·v1–5)
- 데모 저장소: LocalStorage (+BroadcastChannel 탭 동기화) / 운영: Google Apps Script + Sheets
- 배포: 정적 파일 호스팅 어디든 가능 (GitHub Pages, 내부 웹서버 등)
